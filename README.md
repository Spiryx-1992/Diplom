# Инструкция по начальной настройке конфигурации 1С для абонентского обслуживания и расчёта зарплаты

## Заполнение констант

В режиме «1С: Предприятие» заполните следующие константы:
- ВКМ_ТокенУправленияТелеграмБотом — вставьте токен, полученный от @BotFather для вашего Telegram бота.
- ВКМ_ИдентификаторГруппыДляОповещения — укажите chat_id группы/канала, куда бот будет отправлять уведомления (может быть отрицательным числом для групп).
- ВКМ_НоменклатураАбонентскаяПлата — выберите из справочника номенклатуры позицию, которая будет использоваться как «Абонентская плата» в документах обслуживания.
- ВКМ_НоменклатураРаботыСпециалиста — выберите позицию номенклатуры для работ специалистов (почасовая/фиксированная ставка).

Проверка: откройте любой документ «Реализация товаров и услуг» и убедитесь, что номенклатура подставляется автоматически или доступна для выбора.

## Настройка ролей и прав доступа

В режиме «1С:Предприятие» в разделе «Администрирование - «Настройка пользователей и прав» - «Профили групп доступа» проверьте наличие следующих ролей:
- Специалист;
- Менеджер;
- БухгалтерИТФирмы;
- Кадровик-Расчетчик.

По необходимости назначьте их пользователям. 

Проверьте, что вход в базу под каждым типом пользователя соответствует интерфейсу роли.

## Проверка планов видов расчёта и регистров

Убедитесь, что в системе созданы и активны следующие планы видов расчёта:
- ВКМ_ОсновныеНачисления — для окладов, отпусков.
- ВКМ_ДополнительныеНачисления — для премий.
- ВКМ_Удержания — для НДФЛ.

Регистры расчёта, сведений и накопления создаются автоматически при обновлении ИБ. Проверьте в отчётах и документах, что движения по ним формируются корректно.

## Документы

Проверьте доступность и корректность следующих документов:
- ВКМ_ОбслуживаниеКлиента — создание, заполнение, проведение.
- ВКМ_НачисленияЗарплаты, ВКМ_НачислениеПремии, ВКМ_ВыплатаЗарплаты — ввод, расчёт, проведение.
- ВКМ_ГрафикОтпусков — планирование отпусков, контроль пересечений.

## Обработка

- ВКМ_МассовоеСозданиеАктов — запустите и проверьте создание актов.

## Проверка отчётов

Сформируйте каждый из отчётов с тестовыми данными:

- ВКМ_ВыработкаСпециалистов;
- ВКМ_АнализВыставленныхАктов;
- ВКМ_РасчётыССотрудниками;
- ВКМ_РасходЗапланированныхОтпусков.

Убедитесь, что отчёты строятся только по данным регистров.

## Интеграция с Telegram ботом

1.  Убедитесь, что константы с токеном и chat_id заполнены.
2.  Создайте тестовое событие (например, новое обслуживание или закрытие обслуживания).
3.  Проверьте, что бот отправляет уведомление об изменении документа «ВКМ_ОбслуживаниеКлиента» в справочник «ВКМ_УведомленияТелеграмБоту».
   
## Автоматизированное тестирование (Vanessa Automation)

В составе поставки реализованы 4 сценария на Vanessa Automation:

1.	Планирование нескольких обслуживаний.
2.	Закрытие обслуживаний.
3.	Создание актов.
4.	Формирование отчёта.

Запустите сценарии для проверки корректности работы механизмов.
